# บทที่ 1 {#บทที่-1}

บทนำ

## 1.1 ที่มาและความสำคัญ {#1.1-ที่มาและความสำคัญ}

ในปัจจุบัน การบริหารเวลาและการสร้างวินัยในการทำงานหรือการเรียนถือเป็นทักษะสำคัญที่ส่งผลโดยตรงต่อความสำเร็จของนักเรียน นักศึกษา และคนทำงานทั่วไป ผู้ใช้จำนวนมากอาศัยแอปพลิเคชันจับเวลาแบบ Pomodoro หรือแอปจับเวลาโฟกัสทั่วไปเพื่อช่วยแบ่งช่วงเวลาทำงานและพักผ่อน อย่างไรก็ตาม แอปพลิเคชันเหล่านี้ส่วนใหญ่มีเพียงฟังก์ชันจับเวลาพื้นฐาน ขาดกลไกที่สร้างแรงจูงใจให้ผู้ใช้กลับมาใช้งานอย่างต่อเนื่อง

นอกจากนี้ การโฟกัสด้วยตนเองเพียงลำพังโดยไม่มีการติดตามพัฒนาการหรือแรงกระตุ้นจากภายนอกมักส่งผลให้ผู้ใช้ขาดความต่อเนื่องในการสร้างนิสัย เช่น เลิกใช้งานหลังผ่านไปเพียงไม่กี่วัน ไม่เห็นความคืบหน้าของตนเองในระยะยาว และไม่มีบรรยากาศการแข่งขันหรือให้กำลังใจร่วมกับผู้อื่นที่มีเป้าหมายเดียวกัน ส่งผลให้การสร้างวินัยในการโฟกัสของผู้ใช้ไม่ประสบผลสำเร็จเท่าที่ควร และผู้ใช้จำนวนมากล้มเลิกความตั้งใจไปกลางคัน

จากปัญหาดังกล่าว ผู้พัฒนาจึงตระหนักถึงความสำคัญของการนำเทคโนโลยีมาประยุกต์ใช้เพื่อแก้ไขข้อจำกัดที่เกิดขึ้น จึงได้พัฒนาแอปพลิเคชันจับเวลาโฟกัสในรูปแบบเกม (Focus) เพื่อใช้กลไกเกม (Gamification) เช่น ระบบอวตาร พลังชีวิต ค่าประสบการณ์ และไอเทมสะสม มาสร้างแรงจูงใจให้ผู้ใช้โฟกัสอย่างต่อเนื่อง โดยแอปพลิเคชันดังกล่าวจะช่วยให้ผู้ใช้เห็นพัฒนาการของตนเองอย่างเป็นรูปธรรม รวมทั้งช่วยให้ผู้ใช้สามารถโฟกัสร่วมกับเพื่อนผ่านห้องโฟกัสเพื่อสร้างบรรยากาศการแข่งขันและให้กำลังใจซึ่งกันและกัน

## 1.2 วัตถุประสงค์ {#1.2-วัตถุประสงค์}

1.2.1   เพื่อพัฒนาแอปพลิเคชันจับเวลาโฟกัสในรูปแบบเกม Focus ตามหลักการ Pomodoro ที่อยู่ในรูปแบบ community app โดยรองรับผู้ใช้งาน 3 กลุ่มหลัก ได้แก่ ผู้ใช้ทั่วไป (Guest) สมาชิก (Member) และผู้ดูแลระบบ (Admin)

## 1.3 ขอบเขตของโครงงาน {#1.3-ขอบเขตของโครงงาน}

### 1.3.1 ผู้ใช้ทั่วไป (Guest)

1. สมัครสมาชิกได้

### 1.3.2 สมาชิก (Member)

1. เข้าสู่ระบบได้
2. สร้างอวตารเริ่มต้นได้
3. เพิ่มรายการสิ่งที่ต้องการโฟกัสได้
4. ตั้งค่าระยะเวลาโฟกัสและระยะเวลาพักส่วนตัวได้
5. เริ่มจับเวลาโฟกัสในรูปแบบต่อสู้กับบอสได้
6. ยกเลิกเซสชันโฟกัสกลางทางได้
7. กดจบช่วงพักก่อนเวลาที่กำหนดได้
8. เปิดกล่องสุ่มเพื่อรับไอเทมได้
9. ผสมไอเทมเพื่ออัปเกรดเป็นไอเทมที่หายากขึ้นได้
10. ใช้ไอเทมโล่เวลาเพื่อป้องกันพลังชีวิตและสถิติต่อเนื่อง (Streak) ได้
11. เรียกดูและใช้งานไอเทมในกระเป๋าไอเทมได้
12. ทำภารกิจชุบชีวิตอวตารเมื่อพลังชีวิตหมดได้
13. สร้างห้องโฟกัสและเชิญผู้อื่นเข้าร่วมได้
14. เข้าร่วมห้องโฟกัสของผู้อื่นได้
15. เรียกดูเวลาโฟกัสและอันดับของสมาชิกภายในห้องได้
16. ทำเครื่องหมายปิดเป้าหมายของห้องโฟกัสที่ตนเองสร้างได้
17. เพิ่มเพื่อนและเรียกดูสถิติของเพื่อนได้
18. เรียกดูประวัติและสถิติการโฟกัสของตนเองได้

### 1.3.3 ผู้ดูแลระบบ (Admin)

1. เข้าสู่ระบบได้
2. จัดการข้อมูลไอเทมและอัตราดรอปของกล่องสุ่มได้
3. จัดการรูปแบบอวตารเริ่มต้นได้
4. ระงับและปลดระงับบัญชีผู้ใช้ได้
5. เรียกดูแดชบอร์ด (Dashboard) สถิติการใช้งานระบบได้

## 1.4 ประโยชน์ที่คาดว่าจะได้รับ {#1.4-ประโยชน์ที่คาดว่าจะได้รับ}

1.4.1   ผู้ใช้ทั่วไปสามารถสมัครสมาชิกและเริ่มต้นใช้งานแอปพลิเคชันได้อย่างสะดวก
1.4.2   สมาชิกได้รับแรงจูงใจในการโฟกัสอย่างต่อเนื่องผ่านกลไกเกม เช่น ระบบเลเวล พลังชีวิต และไอเทมสะสม ซึ่งช่วยให้สามารถสร้างวินัยในการโฟกัสได้ในระยะยาว
1.4.3   สมาชิกสามารถติดตามพัฒนาการของตนเองและได้รับแรงกระตุ้นจากการโฟกัสร่วมกับเพื่อนในห้องโฟกัส ทำให้เกิดความรู้สึกมีส่วนร่วมและไม่โดดเดี่ยวในการสร้างวินัย
1.4.4   ระบบช่วยลดปัญหาการขาดความต่อเนื่องในการโฟกัส และเพิ่มประสิทธิภาพในการบริหารเวลาของผู้ใช้งาน

## 1.5 เครื่องมือที่ใช้ในการพัฒนา (Development Tools) {#1.5-เครื่องมือที่ใช้ในการพัฒนา-(development-tools)}

> หมายเหตุ: เนื่องจากเอกสาร use case ที่ใช้อ้างอิงไม่ได้ระบุเทคโนโลยีที่จะใช้พัฒนาไว้ รายการด้านล่างเป็นตัวเลือกที่เหมาะสมสำหรับแอปพลิเคชันลักษณะนี้ ผู้พัฒนาควรปรับแก้ให้ตรงกับเครื่องมือที่ใช้งานจริง

### 1.5.1 ฮาร์ดแวร์ (Hardware)

1. เครื่องคอมพิวเตอร์ส่วนบุคคลสำหรับพัฒนาระบบ
2. สมาร์ตโฟนสำหรับทดสอบการทำงานของแอปพลิเคชัน

### 1.5.2 ซอฟต์แวร์ (Software)

1. Visual Studio Code ใช้เป็นเครื่องมือสำหรับการพัฒนาแอปพลิเคชัน
2. Figma ใช้ในการออกแบบส่วนติดต่อผู้ใช้งาน (UI/UX)
3. Postman ใช้ในการทดสอบการทำงานของ API
4. Git และ GitHub ใช้ในการควบคุมเวอร์ชันของซอร์สโค้ด
5. Flutter Framework เป็นเฟรมเวิร์กโอเพนซอร์สสำหรับพัฒนาแอปพลิเคชันบนอุปกรณ์เคลื่อนที่แบบข้ามแพลตฟอร์ม

### 1.5.3 ภาษาโปรแกรมที่ใช้

1. Dart ใช้พัฒนาส่วนหน้า (Frontend) ของแอปพลิเคชันด้วย Flutter Framework
2. JavaScript ใช้พัฒนาส่วนหลัง (Backend) และระบบประมวลผลด้วย Node.js

### 1.5.4 ฐานข้อมูลที่ใช้

1. Firebase Firestore ใช้เป็นฐานข้อมูลสำหรับจัดเก็บข้อมูลต่าง ๆ ของระบบแบบเรียลไทม์
