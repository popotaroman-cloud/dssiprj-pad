## M1 · บัญชีและการเริ่มต้นใช้งาน

### UC-01 · การสมัครสมาชิก

| หัวข้อ | รายละเอียด |
|-------|-----------|
| **รหัสอ้างอิง** | UC-01 |
| **Actor หลัก** | Guest |
| **Actor รอง** | ระบบฐานข้อมูลผู้ใช้ |
| **คำอธิบายโดยย่อ** | Guest สร้างบัญชีผู้ใช้ใหม่เพื่อเปลี่ยนสถานะเป็นสมาชิก ก่อนจะสามารถเข้าถึงฟีเจอร์อื่นใดของแอปได้ นี่คือ use case เดียวที่ Guest ทำได้ |
| **เงื่อนไขก่อนเริ่มทำงาน (Pre-condition)** | ผู้ใช้ยังไม่มีบัญชีในระบบ (สถานะเป็น Guest) และติดตั้งแอปเรียบร้อยแล้ว |
| **ลำดับเหตุการณ์หลัก (Basic Flow)** | 1. Guest กดปุ่ม "สมัครสมาชิก"<br>2. ระบบแสดงฟอร์มกรอกข้อมูล<br>3. Guest กรอกข้อมูลครบถ้วนและกดยืนยัน<br>4. ระบบตรวจสอบความถูกต้องของข้อมูล<br>5. ระบบสร้างบัญชีใหม่ เปลี่ยนสถานะเป็นสมาชิก และพาไปหน้าสร้างอวตาร |
| **ลำดับเหตุการณ์ทางเลือก (Alternative/Exception Flow)** | ข้อมูลไม่ครบ/ไม่ถูกรูปแบบ → แจ้งเตือนให้แก้ไข; อีเมล/ชื่อผู้ใช้ถูกใช้แล้ว → แจ้งเตือนให้แก้ไข; กดยกเลิกกลางคัน → ล้างข้อมูล กลับหน้าแรก ยังเป็น Guest |
| **เงื่อนไขหลังทำงานสำเร็จ (Post-condition)** | สร้างบัญชีใหม่และบันทึกลงฐานข้อมูล สถานะเปลี่ยนจาก Guest เป็นสมาชิก และพาไปยัง UC-02 |
| **ความสัมพันธ์กับ Use Case อื่น** | `<<include>>` UC-02 (บัญชีใหม่ทุกบัญชีต้องสร้างอวตารเสมอ) |
| **เกี่ยวข้องกับ Service/Endpoint อื่น** | - |

## M2 · เซสชันโฟกัส (Core Loop)

### UC-04 · เริ่มจับเวลาโฟกัส (ตีบอส)

| หัวข้อ | รายละเอียด |
|-------|-----------|
| **รหัสอ้างอิง** | UC-04 |
| **Actor หลัก** | สมาชิก |
| **Actor รอง** | ระบบจับเวลา, ระบบบอส |
| **คำอธิบายโดยย่อ** | สมาชิกเลือกรายการที่จะโฟกัสและตั้งเวลา ระบบจำลองเป็นการต่อสู้กับบอสที่มี HP ลดลงตามเวลาที่ผ่านไป ครอบคลุมวงจรทั้งหมดของเซสชันโฟกัสเดียว ตั้งแต่เริ่มจับเวลา ไปจนจบเซสชัน (สำเร็จหรือยกเลิกกลางทาง) รับรางวัล และเข้าสู่ช่วงพัก (เดิมแยกเป็น UC-06, UC-07, UC-09 — ยุบรวมแล้วตามนิยามข้อ 0 ของไกด์ เพราะไม่มี actor ริเริ่มเอง) |
| **เงื่อนไขก่อนเริ่มทำงาน (Pre-condition)** | มีรายการสิ่งที่โฟกัสอย่างน้อย 1 รายการ (UC-03) และไม่มีเซสชันอื่นกำลังทำงานอยู่ |
| **ลำดับเหตุการณ์หลัก (Basic Flow)** | 1. เลือกรายการที่จะโฟกัสและตั้งระยะเวลา (หรือใช้ค่าเริ่มต้นจาก UC-24)<br>2. ระบบตรวจสอบ/คำนวณ HP ล่าสุดของอวตารตามเวลาที่ผ่านไป (ดู Business Rules) ก่อนเริ่ม<br>3. กดปุ่มเริ่มจับเวลา ระบบสร้างบอสด้วย HP = เวลาที่ตั้ง × 100<br>4. แสดงนาฬิกาถอยหลังคู่กับแถบเลือดบอส พร้อม checkpoint ทุก 5 นาที ลดค่าตามเวลาจริง<br>5. เมื่อเวลาครบ (บอส HP=0): ถ้าเซสชัน ≥ 20 นาที คำนวณ EXP และกล่องสุ่มตามความยาวเซสชัน (อ้างอิงตาราง UC-10), เพิ่ม EXP ให้อวตาร (ถึงเกณฑ์ก็เพิ่มเลเวลอัตโนมัติทันที — ดู Business Rules), เติม HP คืนตามสัดส่วน และอัพเดต streak ไฟ (ดู Business Rules)<br>6. บันทึกเซสชันสำเร็จลงประวัติ ระบบเริ่มนับเวลาพักอัตโนมัติ แสดงเวลาที่เหลือ<br>7. เมื่อเวลาพักครบ ระบบแจ้งเตือนให้เริ่มเซสชันถัดไป ถ้าไม่เริ่ม ระบบนับเวลาที่เกินและบันทึกไว้เป็นสถิติ (ไม่กระทบ HP/streak) |
| **ลำดับเหตุการณ์ทางเลือก (Alternative/Exception Flow)** | กดค้างยกเลิกกลางทาง → ส่งต่อ UC-05 (extend); กดเสร็จสิ้นช่วงพักก่อนเวลา → ส่งต่อ UC-08 (extend); ไม่ได้เลือกรายการ → แจ้งเตือน; เซสชัน &lt; 20 นาที → ไม่ได้ EXP/กล่องสุ่ม แต่ยังเติม HP/streak ตามปกติ; HP ที่คำนวณได้ในขั้นตอนที่ 2 เท่ากับ 0 → อวตารเข้าสถานะหมดสติ ล็อกฟีเจอร์บางส่วน ส่งต่อ UC-17.1 |
| **เงื่อนไขหลังทำงานสำเร็จ (Post-condition)** | เซสชันถูกบันทึกลงประวัติ (สำเร็จ/ยกเลิกกลางทาง); รางวัล/HP/streak/เลเวลอัพเดตตามกฎทางธุรกิจ; ช่วงพักเริ่มหรือจบแล้วตามการกระทำของสมาชิก |
| **กฎทางธุรกิจที่เกี่ยวข้อง (Business Rules)** | อัตราแปลงเวลาเป็น HP บอส: 1 นาที = 100 HP<br>ต้องโฟกัสจริงอย่างน้อย 20 นาทีจึงได้ EXP และกล่องสุ่ม<br>อัตราดรอปกล่องสุ่มอิงตามความยาวเซสชัน — ดูตารางใน UC-10<br>เกณฑ์ EXP ต่อเลเวลเพิ่มขึ้นตามลำดับเลเวล EXP เกินหลายเลเวลพร้อมกันจะเพิ่มเลเวลต่อเนื่อง (เดิม UC-14)<br>HP อวตารคำนวณแบบ lazy evaluation จากเวลาที่ผ่านไปนับจาก `last_focus_completed_at`: 0-24 ชม.=เต็ม 100%, 1-2 วัน=70-85%, 2-3 วัน=15-35%, 3.5-4 วัน=0% — ถ้ามีไอเทมโล่เวลา (UC-13) ที่ยังไม่หมดอายุ ข้ามการคำนวณลดทั้งหมด — ถ้า HP=0 อวตารเข้าสถานะ "หมดสติ" ล็อกฟีเจอร์ที่ต้องใช้อวตารแข็งแรง (ห้องโฟกัส UC-18 ถึง UC-20) จนกว่าจะชุบชีวิตสำเร็จ (UC-17.1) (เดิม UC-15, UC-17)<br>Streak ไฟนับแยกจาก HP: โฟกัสสำเร็จอย่างน้อย 1 ครั้ง/วันต่อเนื่องจะเพิ่ม streak, ขาดจะรีเซ็ตเป็น 0 เว้นแต่มีโล่เวลาป้องกันอยู่ (ไม่กระทบ HP หรืออวตารโดยตรง) (เดิม UC-16)<br>เวลาเบรกเกิน (พักเกินกำหนด) เป็นเพียงข้อมูลสถิติ ไม่กระทบ HP/streak โดยตรง (เดิม UC-09) |
| **ความสัมพันธ์กับ Use Case อื่น** | ถูก `<<extend>>` โดย UC-05, UC-08; ถูก `<<include>>` โดย UC-17.1 (ภารกิจชุบชีวิตใช้กลไกเดียวกัน) |
| **เกี่ยวข้องกับ Service/Endpoint อื่น** | - |

### UC-05 · ยกเลิกเซสชันโฟกัสกลางทาง

| หัวข้อ | รายละเอียด |
|-------|-----------|
| **รหัสอ้างอิง** | UC-05 |
| **Actor หลัก** | สมาชิก |
| **Actor รอง** | ระบบคำนวณรางวัลบางส่วน |
| **คำอธิบายโดยย่อ** | สมาชิกยกเลิกเซสชันโฟกัสก่อนเวลาครบกำหนด โดยกดค้างเพื่อยืนยันการยกเลิก (อย่างน้อย 2 วินาที) |
| **เงื่อนไขก่อนเริ่มทำงาน (Pre-condition)** | มีเซสชันโฟกัสกำลังทำงานอยู่ (UC-04) |
