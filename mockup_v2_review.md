# รายงานตรวจสอบความสอดคล้อง — mockup_v2.html

**ขอบเขต:** เทียบ `mockup_v2.html` กับ (1) `generator/guide/mockup_generate_guide.md` (template) (2) `proposal/usecase/usecase_description.md` (26 UC) (3) `usecase_module1-7.puml` + `usecase_overview.puml` (4) `activity_module*_uc*.puml` (5) `proposal/usecase/ui_state_diagram.puml` — **ยังไม่แก้ไขใดๆ ตามคำขอ เป็นรายงานตรวจสอบอย่างเดียว**

**สรุปผล:** โครงสร้างหน้าจอครอบคลุม 24 จาก 26 use case, business logic ส่วนใหญ่ (HP/EXP/กล่องสุ่ม/Pity/ผสมไอเทม/ชุบชีวิต) ตรงตามเอกสารละเอียดมาก แต่พบ **1 use case หายไปทั้งหมด**, **2 ฟีเจอร์ที่ไม่มี use case รองรับ (นอกขอบเขต)**, **1 จุดที่ business rule ขัดแย้งกับเอกสารต้นทางโดยตรง**, และ **ช่องโหว่ validation หลายจุด** (ไม่เช็คขอบเขตค่า/เงื่อนไขตาม Alt Flow)

## ปัญหาหลัก

### 1. UC-26 (ตั้งค่าการแจ้งเตือน เสียง/การสั่น) หายไปทั้งหมด
ไม่มี screen case ใดๆ สำหรับ UC-26 เลย (`renderScreen()` บรรทัด 1280-1317 ไม่มี `case "notifSettings"`) และค้นทั้งไฟล์ไม่พบคำว่า "เสียง"/"สั่น"/"แจ้งเตือน" ในบริบทของหน้าตั้งค่าเลย หน้า `screenSettings()` (บรรทัด 1533) มีแค่ปุ่มแก้ไขโปรไฟล์/เปลี่ยนรหัสผ่าน/ส่ง Feedback/ออกจากระบบ ไม่มีเมนู "การแจ้งเตือน" ตามที่ `ui_state_diagram.puml` กำหนดไว้ (state `NotifSettings`)

### 2. ฟีเจอร์ "จัดการรูปแบบอวตาร" (Admin) ไม่มี use case รองรับ — ขัดกับเอกสารที่ระบุชัดว่าตัดออกจากขอบเขตแล้ว
`screenAdminAvatars()` (บรรทัด 2078) และปุ่มในแดชบอร์ดผู้ดูแล (บรรทัด 2041) เป็นฟีเจอร์เต็มรูปแบบ (เพิ่ม/เปิด-ปิดใช้งานรูปแบบอวตาร) แต่ `usecase_description.md` ไม่มี use case นี้เลย และ `ui_state_diagram.puml` เขียนไว้ชัดเจนในหมายเหตุ (บรรทัด 189): "**ไม่มี use case จัดการรูปแบบอวตารอีกต่อไป (ตัดออกจากขอบเขต)**" — mockup กลับพาไปคนละทิศกับเอกสารสองฉบับนี้

### 3. ฟีเจอร์ "ลืมรหัสผ่าน" ไม่มี use case รองรับ
Flow `forgot` → `forgotSent` → `resetPw` (บรรทัด 919-930) ไม่มี use case ใดกล่าวถึงเลย — UC-08 (เข้าสู่ระบบ) มี Alt Flow เดียวคือ "อีเมล/รหัสผ่านไม่ถูกต้อง → แจ้งเตือน ไม่ออก token" เท่านั้น ไม่มีการลืมรหัสผ่านแบบ self-service ที่เอกสารรองรับ

### 4. Business Rule ยกเลิกเซสชันกลางทาง (UC-07) ขัดกับ usecase_description.md โดยตรง
`usecase_description.md` (บรรทัด 130 และ 133) ย้ำ **สองครั้ง** ว่า: "HP ลดลง 10 หน่วย **(ค่าคงที่ ไม่คำนวณตามสัดส่วนเวลาที่โฟกัสไปแล้ว)**"

แต่ `cancelSessionConfirmed()` (บรรทัด 1064-1076) ทำตรงข้าม — **คืน HP ตามสัดส่วนเวลาที่โฟกัสไปแล้ว**:
```js
const heal = Math.round(member.maxHp * (elapsedMin / s.focusMin));
member.hp = clamp(member.hp + heal, 0, member.maxHp);
```
พร้อม toast ข้อความ "คืน HP ตามสัดส่วน ไม่ได้กล่องสุ่ม" — จุดนี้ mockup สอดคล้องกับ `ui_state_diagram.puml` ฉบับปัจจุบัน (บรรทัด 114-115, 121-123 ก็เขียนว่า "คืน HP สัดส่วน" เหมือนกัน) แปลว่า mockup กับ ui_state_diagram ไปด้วยกัน แต่ **ทั้งคู่ขัดกับ usecase_description.md** ซึ่งเป็นเอกสารที่ระบุกฎนี้อย่างเจาะจงและย้ำซ้ำ ต้องเลือกว่าเอกสารไหนถูกแล้วปรับอีกฝั่งให้ตรงกัน (ไม่ใช่ปล่อยขัดกันไว้)

### 5. "หลุดโฟกัสนอกการควบคุม" (Interruption) ไม่มีเกณฑ์ 2 นาทีเลย
UC-07 Alt Flow ระบุว่า: ถ้ากลับเข้าแอปภายใน 2 นาทีให้นับเวลาต่อ แต่ถ้าเกิน 2 นาทีให้ถือว่า "ออกกลางคัน" (HP -10)

`visibilitychange` handler (บรรทัด 872-883) ไม่เช็คระยะเวลาที่หายไปเลย — ไม่ว่าจะออกจากแอปไป 10 วินาทีหรือ 10 ชั่วโมง ระบบจะเลื่อนเวลาเริ่มเซสชันชดเชยให้เสมอโดยไม่มีบทลงโทษใดๆ:
```js
if (document.hidden) { s._pauseSimSnapshot = simNow(); }
else if (s._pauseSimSnapshot != null) {
  const gap = simNow() - s._pauseSimSnapshot;
  s.phaseStartedSim += gap;  // เลื่อนให้เสมอ ไม่มีเงื่อนไข 2 นาที
  ...
```

### 6. ปุ่ม "เริ่มห้อง" ไม่เช็คสมาชิกขั้นต่ำ 2 คน
UC-16 Alt Flow ระบุว่าปุ่ม "เริ่มห้อง" ต้องถูก**ปิดใช้งาน**ถ้ามีสมาชิกไม่ถึง 2 คน แต่ปุ่มในบรรทัด 1888 (`screenRoomView`) ไม่มีเงื่อนไข `disabled` ใดๆ และ `startRoom()` (บรรทัด 811-816) ก็ไม่เช็คจำนวนสมาชิกก่อนเริ่มเช่นกัน — เจ้าของห้องคนเดียวกดเริ่มห้องได้ตลอด

### 7. Ranking ไม่มี Tie-break ตามเวลาพัก
`closeRoom()` (บรรทัด 827-838) จัดอันดับด้วย `room.members.slice().sort((a, b) => b.focusSec - a.focusSec)` เท่านั้น ไม่มีการเทียบเวลาพัก (Rest) เมื่อเวลาโฟกัสสะสมเท่ากัน ทั้งที่ Business Rules ของ UC-16 ระบุ Tie-break ไว้ชัดเจน ("ผู้ที่มีเวลาพักสะสมน้อยกว่าเป็นผู้ชนะ")

### 8-9. ขอบเขตค่า Focus/Break/Room Duration ไม่ตรงหรือไม่ถูกเช็ค
- `saveDefaultDuration` (บรรทัด 972, UC-05): จำกัดที่ **1-180 นาที** สำหรับ Focus แต่เอกสารระบุ **1-600 นาที**
- `beginSession` (บรรทัด 1029-1031, UC-07): เช็คแค่ `< 1` ไม่มี upper bound เลย (เอกสาร: Focus ≤600, Break ≤60)
- `createRoomSubmit` (บรรทัด 1130-1134, UC-16): เช็คแค่ "กรอกครบ" ไม่มีการเช็คช่วง Room Duration 5-1000 นาที หรือ Focus/Break 1-600/1-60 เลย

## จุดที่ตรวจแล้วตรงกันดี

- **UC-07 (ส่วนที่เหลือ):** Base Max HP=100, Level Scaling +5/เลเวล (`calcMaxHp`), HP Decay 2 หน่วย/ชม. (`settleDecay`), สูตร EXP = นาที×2, เพดานเลเวล — ตรงตามเอกสาร
- **UC-10 (กล่องสุ่ม):** ช่วงความยาวเซสชันกำหนดชนิดกล่อง (20-44/45-89/90+ → bronze/silver/gold) และอัตราดรอปทั้ง 3 ตาราง ตรงตามเอกสารทุกตัวเลข, Pity System แยกตัวนับตามชนิดกล่อง เกณฑ์ครบ 10 กล่อง (`pityCount >= 9`) ตรงตามเอกสาร
- **UC-11 (ผสมไอเทม):** จำนวนที่ใช้ (5 ชิ้นอัพเกรด, 3 ชิ้นผสมโล่เวลา), แยกสูตรโล่ 24 ชม. (สามัญ/หายาก) vs 48 ชม. (หายากมาก/ตำนาน) ตรงตามเอกสาร
- **UC-15 (ชุบชีวิต):** ภารกิจ 20 นาทีคงที่, ลดเลเวล 2 ระดับ (ต่ำสุด Lv.1), รีเซ็ต EXP=0, คำนวณ Max HP ใหม่ตาม Level Scaling — ตรงตามเอกสาร
- **UC-08 (Streak):** เพิ่ม 1 ระดับเมื่อ login ครั้งแรกของวัน, รีเซ็ตเป็น 0 เมื่อขาด login เกิน 1 วันเต็ม (เว้นแต่มีโล่เวลาคุ้มครอง) — ตรงตามเอกสาร
- **UC-24/UC-25 (Admin):** จัดการอัตราดรอป 3 ชุดแยกตามชนิดกล่อง, ตรวจผลรวม = 100%, จัดการสูตรผสม, ดู/ตอบกลับ Feedback — ตรงตามเอกสารดี
- **Role-based navigation:** ครบ 3 role (Guest/Member/Admin) ตรงกับ Actor ใน `usecase_overview.puml`, มีปุ่มสลับ role ตามคู่มือ §5
- **Screen coverage:** มีหน้าจอรองรับครบ 24 จาก 26 UC (ขาดเฉพาะ UC-26 ตามข้อ 1)

## ข้อเสนอแนะ (เรียงตามความสำคัญ — ยังไม่ได้แก้ตามที่ขอ)

1. เพิ่มหน้า/ส่วนตั้งค่าการแจ้งเตือน (UC-26) ที่หายไปทั้งหมด
2. **ตัดสินใจก่อน** ว่าโทษยกเลิกเซสชันกลางทางคือ "-10 คงที่" (ตาม usecase_description.md) หรือ "คืน HP ตามสัดส่วน" (ตามที่ ui_state_diagram+mockup ทำอยู่) แล้วแก้เอกสารฝั่งที่ผิดให้ตรงกันทั้งชุด — ไม่ใช่แก้ mockup เฉยๆ โดยไม่เช็คว่าเอกสารไหนคือ source of truth
3. เพิ่มเกณฑ์ 2 นาทีในกลไก "หลุดโฟกัส" (visibilitychange)
4. เพิ่มการเช็ค/ปิดปุ่ม "เริ่มห้อง" เมื่อสมาชิกไม่ถึง 2 คน
5. เพิ่ม tie-break ตามเวลาพักใน `closeRoom()`
6. แก้ขอบเขตค่า Focus/Break ให้ตรง (1-600/1-60) ทั้ง 3 จุด (UC-05 default, UC-07 ตั้งเซสชัน, UC-16 ตั้งห้อง 5-1000 นาที)
7. พิจารณาตัด/ย้ายฟีเจอร์ "ลืมรหัสผ่าน" และ "จัดการรูปแบบอวตาร (Admin)" ออก เพราะไม่มี use case รองรับ — หรือถ้าต้องการคงไว้จริง ต้องเพิ่ม use case ใหม่ในเอกสารทั้งชุดให้ตรงกันก่อน
8. แก้คอมเมนต์ในโค้ดที่อ้างเลข UC ผิด (บรรทัด 582, 1676, 2123)
